# IPv6 WireGuard Manager - 快速入门指南

## 🚀 5分钟快速部署

### 第一步：一键安装
```bash
curl -fsSL https://raw.githubusercontent.com/ipzh/ipv6-wireguard-manager/main/install.sh | bash
```

### 第二步：访问系统
安装完成后，通过以下地址访问：
- **IPv4**: `http://您的服务器IP`
- **IPv6**: `http://[您的服务器IPv6地址]`

### 第三步：登录系统
- **用户名**: `admin`
- **密码**: `admin123`

### 第四步：修改密码
1. 点击右上角用户头像
2. 选择"系统设置"
3. 在"修改密码"标签页中更改密码

## 🎯 核心功能快速上手

### 1. 创建WireGuard服务器

#### 快速创建
1. 进入 **服务器管理** 页面
2. 点击 **"添加服务器"** 按钮
3. 填写基本信息：
   - 名称：`My VPN Server`
   - 端口：`51820`
   - IPv4地址：`10.0.0.1/24`
   - IPv6地址：`fd00::1/64`
4. 点击 **"保存"** 完成创建

#### 启动服务器
1. 在服务器列表中点击 **"启动"** 按钮
2. 等待状态变为 **"运行中"**

### 2. 添加客户端

#### 快速添加
1. 进入 **客户端管理** 页面
2. 点击 **"添加客户端"** 按钮
3. 选择关联的服务器
4. 填写客户端信息：
   - 名称：`My Phone`
   - IPv4地址：`10.0.0.2/32`
   - IPv6地址：`fd00::2/128`
5. 点击 **"保存"** 完成添加

#### 获取配置
1. 在客户端列表中点击 **"QR码"** 按钮
2. 使用手机扫描二维码
3. 或点击 **"下载配置"** 获取.conf文件

### 3. 配置域名和SSL（可选）

#### 使用Let's Encrypt免费SSL
1. 进入 **系统设置** → **域名与SSL**
2. 输入您的域名：`vpn.example.com`
3. 启用 **SSL** 选项
4. 选择 **Let's Encrypt**
5. 配置DNS提供商（如Cloudflare）
6. 输入DNS API密钥
7. 点击 **"生成SSL证书"**

#### 使用自定义证书
1. 启用 **SSL** 选项
2. 选择 **自定义证书**
3. 粘贴您的SSL证书内容
4. 粘贴您的私钥内容
5. 点击 **"保存域名设置"**

## 📱 客户端连接指南

### Android客户端
1. 从Google Play下载 **WireGuard** 应用
2. 打开应用，点击 **"+"** 按钮
3. 选择 **"扫描二维码"** 或 **"从文件导入"**
4. 扫描管理界面生成的QR码或导入.conf文件
5. 点击连接按钮开始使用

### iOS客户端
1. 从App Store下载 **WireGuard** 应用
2. 打开应用，点击 **"+"** 按钮
3. 选择 **"扫描二维码"** 或 **"从文件导入"**
4. 扫描管理界面生成的QR码或导入.conf文件
5. 点击连接按钮开始使用

### Windows客户端
1. 从官网下载 **WireGuard** 客户端
2. 安装并打开客户端
3. 点击 **"添加隧道"** → **"从文件添加"**
4. 选择下载的.conf文件
5. 双击隧道名称开始连接

### Linux客户端
1. 安装WireGuard：
   ```bash
   # Ubuntu/Debian
   sudo apt install wireguard
   
   # CentOS/RHEL
   sudo yum install wireguard-tools
   ```
2. 复制配置文件：
   ```bash
   sudo cp client.conf /etc/wireguard/wg0.conf
   ```
3. 启动连接：
   ```bash
   sudo wg-quick up wg0
   ```

## 🛠️ 故障排除

### 客户端无法连接
1. **检查服务器状态**：确保服务器正在运行
2. **检查防火墙**：确保端口51820已开放
3. **检查配置**：验证客户端配置是否正确
4. **检查网络**：确保服务器网络连接正常

### SSL证书问题
1. **检查域名解析**：确保域名正确解析到服务器IP
2. **检查DNS配置**：验证DNS API密钥是否正确
3. **检查证书状态**：查看证书是否成功生成
4. **检查Nginx配置**：确保SSL配置正确

### 性能问题
1. **检查系统资源**：查看CPU、内存使用情况
2. **检查网络带宽**：确保网络带宽充足
3. **优化配置**：调整MTU、Keepalive等参数
4. **监控日志**：查看系统日志找出问题
