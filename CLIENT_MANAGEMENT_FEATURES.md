# 客户端管理功能详细说明

## 功能概述

IPv6 WireGuard Manager 提供了完整的客户端管理功能，支持单个和批量客户端添加，具备智能地址分配和冲突检测能力。

## 核心功能

### 1. 单个客户端添加 ✅

#### 功能特点
- 支持自动地址分配和手动指定地址
- 自动生成客户端密钥对
- 自动配置服务器端规则
- 生成完整的客户端配置包

#### 使用方法
```bash
# 启动管理器
ipv6-wg-manager

# 选择客户端管理 -> 添加客户端
# 选择地址分配方式：
# 1. 自动分配地址（推荐）
# 2. 手动指定地址
```

#### 自动地址分配
- 系统自动扫描地址池
- 选择第一个可用的IPv4和IPv6地址
- 自动避免地址冲突
- 支持IPv4: 10.0.0.0/24, IPv6: 2001:db8::/48

### 2. 批量客户端添加 ✅

#### 快速批量添加
- **功能**: 一次添加多个客户端，自动分配地址
- **支持**: 自定义客户端数量、名称前缀、起始索引
- **特点**: 完全自动化，无需手动配置地址

```bash
# 示例：添加10个客户端
quick_batch_add_clients 10 "client" 1
# 结果：client1, client2, ..., client10
# 地址：自动分配，避免冲突
```

#### CSV文件批量添加
- **功能**: 使用CSV文件批量添加客户端
- **支持**: 混合使用自动分配和手动指定地址
- **格式**: 客户端名称,IPv4地址,IPv6地址,描述

```csv
# 示例CSV文件
client1,auto,auto,测试客户端1
client2,10.0.0.100/32,2001:db8::100/128,指定地址客户端
mobile1,auto,auto,移动设备1
```

### 3. 智能地址池管理 ✅

#### 地址池状态监控
- **IPv4地址池**: 10.0.0.0/24 (253个可用地址)
- **IPv6地址池**: 2001:db8::/48 (9998个可用地址)
- **实时统计**: 显示已用、可用、使用率

#### 地址冲突检测
- **自动检测**: 添加客户端时自动检测冲突
- **批量检测**: 检查所有客户端的地址冲突
- **冲突报告**: 详细显示冲突的地址和使用次数

#### 地址重新分配
- **功能**: 为现有客户端重新分配地址
- **场景**: 地址冲突、网络变更、客户端迁移
- **特点**: 保持客户端配置，仅更新地址

### 4. 客户端状态管理 ✅

#### 客户端数据库
- **格式**: CLIENT_NAME|PRIVATE_KEY|PUBLIC_KEY|IPV4_ADDRESS|IPV6_ADDRESS|CREATED_DATE|LAST_SEEN|STATUS
- **功能**: 完整的客户端信息记录
- **查询**: 支持按名称、地址、状态查询

#### 连接状态监控
- **实时监控**: 显示客户端连接状态
- **传输统计**: 显示上传/下载流量
- **连接时间**: 记录最后握手时间
- **自动更新**: 自动更新最后连接时间

#### 客户端清理
- **自动清理**: 清理长期不活跃的客户端
- **可配置**: 支持自定义清理时间阈值
- **安全**: 清理前确认，避免误删

### 5. 配置生成和导出 ✅

#### 自动配置生成
- **客户端配置**: 完整的WireGuard配置文件
- **安装脚本**: 自动安装脚本（支持多系统）
- **QR码**: 移动设备扫码配置
- **配置包**: 压缩包包含所有文件

#### 批量导出
- **格式支持**: tar.gz, zip, 单独文件
- **批量打包**: 所有客户端配置打包
- **分类导出**: 按客户端类型分类导出

## 使用场景

### 场景1: 企业VPN部署
```bash
# 1. 快速添加100个员工客户端
quick_batch_add_clients 100 "employee" 1

# 2. 添加管理员客户端（指定地址）
add_client "admin1" "10.0.0.100/32" "2001:db8::100/128"

# 3. 添加访客客户端（自动分配）
add_client "guest1" "auto" "auto"
```

### 场景2: 服务提供商
```bash
# 1. 使用CSV文件批量添加客户
batch_generate_clients "customers.csv" "true"

# 2. 监控客户端连接状态
monitor_client_connections

# 3. 定期清理不活跃客户端
cleanup_inactive_clients 90
```

### 场景3: 开发测试环境
```bash
# 1. 快速创建测试客户端
quick_batch_add_clients 20 "test" 1

# 2. 检查地址池状态
show_address_pool_status

# 3. 测试完成后清理
# 手动删除或使用清理功能
```

## 配置示例

### 客户端配置文件示例
```ini

PrivateKey = CLIENT_PRIVATE_KEY
Address = 10.0.0.2/32, 2001:db8::2/128
DNS = 8.8.8.8, 2001:4860:4860::8888

[Peer]
PublicKey = SERVER_PUBLIC_KEY
Endpoint = SERVER_IP:51820
AllowedIPs = 0.0.0.0/0, ::/0
PersistentKeepalive = 25
```

### 常见问题

#### 1. 地址分配失败
```bash
# 检查地址池状态
show_address_pool_status

# 检查地址冲突
check_all_address_conflicts

# 清理不活跃客户端
cleanup_inactive_clients
```

#### 2. 客户端连接失败
```bash
# 检查客户端配置
get_client_info "client_name"

# 检查服务器配置

# 重新分配地址
reallocate_client_address "client_name"
```

#### 3. 批量添加失败
```bash

cat examples/clients.csv

batch_generate_clients "clients.csv" "true"

show_address_pool_statistics
```
